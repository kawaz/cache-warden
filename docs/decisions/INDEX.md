- [DR-0008](./DR-0008-single-daemon-hosting.md) — 単一デーモンプロセス直担型。`cache-warden run` = 1 プロセス（tokio）で全アダプタを listener task として in-process 直担（決定打は秘密値の 1 プロセス閉じ込め）。管理 CLI ↔ デーモンは control socket（KV socket API と統合、プロトコル詳細は次ステップ）。サービス登録は単一バイナリ + `run`。内部サブコマンド方式・アダプタ別デーモンを却下。DR-0003 / DR-0004 が残したホスティング形態・デーモン境界を確定
- [DR-0009](./DR-0009-control-socket-protocol-v1.md) — control socket プロトコル v1。transport = UDS（0600 / stale 検知 / 二重起動拒否）、framing = JSON Lines、秘密値は base64（`*_b64`）。コマンド = `ping` / `status`（値非露出）/ `kv.set` / `kv.get` / `kv.del` / `kv.list`。peer 認証 = LOCAL_PEERPID / SO_PEERCRED → ancestry を requester として渡す（解釈はまだしない）。Authenticator は AllowAll 暫定配線。length-prefixed バイナリ・gRPC/CBOR・素 JSON 文字列を却下。DR-0008 が残したプロトコル設計を確定
- [DR-0010](./DR-0010-config-and-reauth-command.md) — TOML config と再認証コマンド方式。実 Authenticator = `CommandAuthenticator`（lib、exit 0=承認/非ゼロ=拒否/spawn失敗=Unavailable、AuthContext を env で渡し秘密値は渡さない、timeout なし、warden DR-009 踏襲）。TOML config（CLI、`deny_unknown_fields`、`$CACHE_WARDEN_CONFIG`→`$XDG_CONFIG_HOME`→`~/.config` 探索、`[daemon].socket`/`[auth].command`/`[kv.*]` 起動時プリロード）。**static を config に書けない設計**（平文秘密値の流出防止、書いたら設定エラー）、auth 省略時=AllowAll、socket 優先順位 CLI>config>default、`config show|path|edit` 追加。ビルトイン TouchID 先行・config への static 値許可・config 必須化を却下。DR-0009 の AllowAll 暫定配線を置換
- [DR-0012](./DR-0012-process-access-policy.md) — socket 層プロセスアクセス制御（`allowed_processes`）。socket 層・key 層とも実装済み（key 層 = `[kv.NAME].allowed_processes`、config 専用、kv.get で gate、e2e 済み）/ 空配列=全許可（不変条件）/ 照合=全祖先 OR + 実行ファイル basename 完全一致（glob/regex なし、warden 踏襲）/ `name()==None` の祖先はスキップ / **pid 取得失敗・祖先遡上失敗時は fail-closed（拒否）**（warden は fail-open だが安全側に倒す差異）/ 接続冒頭で 1 回判定し不許可なら全リクエスト SSH_AGENT_FAILURE（列挙も署名も隠す）/ 照合はアダプタ層（DR-0004）/ key 層は socket 層と直列で交差空=全拒否を実装（warden の罠を踏襲しない）。port plan Iteration 5。
- [DR-0021](./DR-0021-signal-shutdown-handling.md) — シグナルベースの shutdown 処理。macOS の `ptrace(PT_DENY_ATTACH)` (DR-0007 の hardening 層) が tokio の非同期シグナル driver を壊す実機確定バグへの対処。SIGINT/SIGTERM をプロセス全スレッドで `pthread_sigmask` ブロックし専用 std スレッドの `sigwait` で同期受信 = kqueue driver を完全バイパス (`block_shutdown_signals` を runtime build 前 + `on_thread_start` で belt-and-suspenders、シグナルスレッドは startup より前に spawn し `Notify` permit で startup 中のシグナルもラッチ) / 全シグナルブロックで OS の即 terminate 安全網が消えるため shutdown watchdog (graceful が `SHUTDOWN_GRACE`=5s 内に完了しなければ control socket unlink + `_exit(0)`、正常時 ~24ms で非発火) で bounded-exit を保証 / signal mask は fork+exec 継承するので子プロセス (source/auth command・op CLI) は `spawn_with_clean_signal_mask` で空マスク起動 = killable 維持 / スレッド spawn 失敗時の縮退経路はシグナルをアンブロックして OS default kill に戻す。tokio::signal 継続・PT_DENY_ATTACH 撤去・async timeout・子 mask 放置を却下。Codex レビューで子 mask 継承と fallback の穴を是正
- [DR-0022](./DR-0022-fetch-failure-backoff.md) — 秘密値 fetch 失敗時の short-term backoff (negative cache)。**[2026-06-14 Codex review v1 + v2 反映、初版から 2 ラウンド改訂]** core `Store` に **failure 状態専用の別フィールド** `failure_backoffs: BTreeMap<String, FailureRecord>` を持たせ (entry 案は DR-0014 entries-only モデルと矛盾)、`Store::regenerate` / `Store::get_or_regenerate` の前段で backoff 期間 (default `5s`、config `[daemon].fetch-failure-backoff`、`"0s"` で無効化) 内の再 fetch を抑止し新 variant `RegenerateOutcome::Backoff` (および `RegenerateDefOutcome::Backoff`) を返す。**`failure_backoffs` の lifetime は `definitions` と一致** (`kv.del --with-define` で削除、`kv.del` 値のみ・hard-ttl 切れでは残す)、**`Store::set` は failure_backoffs を触らない** (副作用なし、操作は regenerate 経路の内部のみ)。**前提条件** = `lazy_load_op_key` を core API 経由に統一する refactor を **separate PR** で先行 (A-3a)、A-3a 遅延中の dogfood 実害は許容して構造を優先 (= 暫定 adapter backoff は不採用)。**分類は TTL/pin と別カテゴリ** = source 実行の retry policy / circuit breaker。`Store::new()` は維持、`set_failure_backoff(Duration)` setter で daemon 側だけ有効化 (= library consumer は default disabled)。process restart 揮発は仕様、restart loop attack は service manager の責務 (cache-warden scope 外)。`status` / 診断ログに `backoff_until` 露出 (可観測性)。離席中の op TouchID 連発 (op-refetch loop) を止める。adapter 層実装・指数 backoff・暫定重複を不採用。失敗種別区別 (案 C) と graceful restart での failure_backoffs 引き継ぎ (Q6) は Open Question
- [DR-0023](./DR-0023-op-discovery-startup-latency.md) — 起動時 op discovery のブロッキングを解消。Phase 1 (Accepted) = `spawn_listeners` / `register_definitions` を `tokio::task::spawn_blocking` でラップ + `select!` で shutdown signal と並行 await、main runtime worker を `Command::output()` の同期ブロックから解放。Phase 2 (Proposed) = `discover_keys` を lazy + background refresh 化、Provider 再設計 (`ssh-agent-provider-architecture` issue) と一体化候補。DR-0008 単一 daemon を維持しつつ DR-0021 watchdog で bounded-exit を再利用。detach / timeout / process 分離を不採用
- [DR-0020](./DR-0020-macos-signing-and-app-bundle.md) — macOS 署名・notarization と .app バンドル。warden 実証済みの 2 層解 (codesign Developer ID + notarize + staple で identity 安定化 / `.app` + `AssociatedBundleIdentifiers` で TCC を Bundle ID ベース化 = brew upgrade のパス変化に耐える。symlink 案は TCC が実体パス解決するため効かない = warden v0.1.11 実証) を移植 / リリース tarball は .app + 素バイナリ両方 / register は .app 内実行を検出して plist に反映 / **secrets はプロダクト別に新規発行** (App-Specific Password が rotate 単位、warden archive 時に独立) / notarization 403 runbook 等のノウハウも docs ごと移植 (warden は archive ルート)。**実装済み (v0.20.0)**: codesign/notarize/staple + .app 生成 + brew cask は release.yml、register の .app 検出→AssociatedBundleIdentifiers 反映は service.rs

- [DR-0018](./DR-0018-typed-sources-auth-and-prefetch.md) — 型付き source / auth スキーマと prefetch モード。`source = "<kind>"` が判別子・kind 別テーブルに実態（`command.argv/cwd/env` / `op.uri/account`。選ばれない kind テーブルは無視 = 排他検証が概念ごと消え、spec を残したまま 1 行切替可。bare `command = []` 配列形は廃止。kind 別 deny_unknown_fields で optional 追加は非破壊）/ **原形保持** = 実行時に argv へ lowering するが定義・status・永続化・冪等比較は型付き形式（op:// 糖衣の原形破棄を解消）/ コアは第 2 の不透明スロット（value 型と source 型は直交軸）/ `[auth]` も type 化（`command` v1、`touchid`・`push` の将来枠 = ビルトイン TouchID の受け皿）/ prefetch は 2 トリガ: 起動時 config（authsock source の `prefetch` opt-in、パターン A = 全鍵 **SoftExpired 封印**で鍵ごとの初回が warden 認証を通る。B = lazy 既定）+ 手動 `kv prefetch [KEY...] [--namespace NS]... - [DR-0017](./DR-0017-kv-namespaces.md) — KV namespace。`--defs` の KEY 衝突問題の解。内部キーは `NS/KEY` 合成（コアは flat のまま、プロトコル境界の概念）/ **KEY/NS 文字種を `[A-Za-z0-9_]+` に絞り生成時強制**（`.` = TOML dotted-key ネスト化 footgun、`-` = inject 終端の丸呑み・jq/env 名非互換、を排除。env 変数風 identifier に統一、DR-0013 文法改訂）/ NS は単一セグメント、デフォルト `"default"` / **CLI の KEY への `ns/key` 埋め込みは拒否**（指定は `--namespace` 一本、排他規則を作らない）/ 参照は `cache-warden://[NS/]KEY` を許可（未修飾 = デフォルト NS に解決、修飾済み = 絶対参照）/ デフォルト NS はフラグ > `CACHE_WARDEN_NAMESPACE` > `[cli].namespace` > default（dry-run 極性と同機構、direnv 連携が本命）/ config・defs は per-entry `namespace` フィールド（あり = 絶対 / なし = 文脈デフォルト、参照と同型規則で排他不要。NS 無し defs は使い回し可）/ authsock keys は `"NS/KEY"` 修飾可（機械向け表面）/ `kv list --all`。CLI への ns/key 埋め込み・手動 prefix 規約・コア (ns,key) キー・階層 NS・`[kv.NS.KEY]` ネストを却下。
- [DR-0016](./DR-0016-otp-value-type.md) — OTP 値型。キャッシュするのは seed（base32 / otpauth:// URI、TTL・mlock・zeroize が全部効く）、6 桁コードは get 毎にデーモン側で導出する派生ビュー（A 案の UX を B 案のレイヤリングで = ハンドラ層変換 1 枚、コアは OTP を知らない）/ **seed は write-only**（デーモンから出ない、クライアントには寿命 30 秒のコードのみ = op より強い）/ `--type otp` 定義メタデータ、digits/period/algorithm は URI < フラグ / `?attribute=otp` source との組合せは define 時エラー（30 秒で死ぬ値のキャッシュ防止）/ 推奨 = seed を op に置き `--source` 定義（lazy regenerate で再起動自己修復）/ hmac+sha1 は CLI crate（コア依存最小は不変）。独立翻訳アダプタ・クライアント側導出・コードのキャッシュ・seed 読出許可・暗号手書きを却下
- [DR-0015](./DR-0015-dry-run-verification-mode.md) — dry-run 検証モードと reveal デフォルトの統一。`kv get` / `run` / `inject` はデフォルト実値（op の「移送 = 実値」と同極性）/ `--dry-run` = 値を出さない full-chain 検証（upstream 実行・再認証・regenerate まで完走、認証ゲート有効、値は応答に載せない = クライアントに届かない）/ マスク値 `<cache-warden:KEY:masked>` / 失敗 `<cache-warden:KEY:failed>`、dry-run は途中 fail-closed せず全評価 + 非ゼロ終了 / デフォルト極性は `[cli].default-mode` と `CACHE_WARDEN_DRY_RUN` で切替（エージェント環境だけ dry-run 既定にできる）/ help に dry-run の存在を明記（エージェント発見性）/ `kv.get` に `dry_run` 追加。非対称デフォルト・全 masked・浅い dry-run・クライアント側マスキングを却下
- [DR-0014](./DR-0014-kv-definition-model.md) — KV definition モデル。動詞の責務分離: `kv define`（定義登録のみ、実行は初回 get まで lazy、冪等 = 完全一致 no-op / 不一致エラー）/ `kv set`（static 専用、`--command` 廃止）/ `kv get`（読み専念、定義があれば regenerate 経路で lazy 生成）。定義レジストリは値ストアと分離（NotLoaded を EntryState に作らない、DR-0004 同型）/ `kv del` = 値のみ破棄、`--with-define` で定義ごと / `--source URI`（v1 は op:// ビルトイン、scheme テーブルは follow-up）/ 定義 4 レイヤ（config lazy 化 + `preload` opt-in + authsock 参照鍵は自動 eager、オンライン定義の永続化 opt-in（値は書かない・config 優先マージ）、`--defs FILE` 一括 define、純オンライン）/ defs 自動探索なし（データ→コード防止）/ 参照クエリ拡張（インライン define）は opt-in 設計で後送り。get-or-init 案・eager define・NotLoaded 追加・declarative 上書き・値永続化を却下。DR-0010 / DR-0013 を一部改訂
- [DR-0013](./DR-0013-secret-reference-injection.md) — secret reference 注入（`run` / `inject`）。参照構文 = `cache-warden://[NS/]KEY`（スキーム 1 種、文字種は DR-0017 で `[A-Za-z0-9_]+` に改訂、再帰展開なし）/ 解決 = control socket `kv.get`（dedup、fail-closed）/ `run` = env のみ注入面（whole-value 規則、op run 互換）+ `--env` + 解決後 exec（親プロセス残置なし、127/126 慣習）、**argv は置換しない**（ps 漏洩防止、警告して verbatim）/ `inject` = テンプレ substring 置換（バイナリ安全、`--out` は 0600）/ 実装は CLI crate に閉じる。argv 置換・出力マスキング・env 部分置換・短縮スキーム・`--env-file` を却下
- [DR-0011](./DR-0011-ttl-base-separation-and-pin.md) — TTL 基準の 2 分離と pin API。`activated_at` を `loaded_at`（set/regenerate 固定 = hard 基準）と `extended_at`（extend で動く = soft 基準）に分離し、「extend し続ければ hard が永遠に来ない」曖昧仕様を解消（extend は soft だけ延命、hard は値の絶対寿命として固定）。明示の `pin`（`pin_until(deadline)`）= 期限まで soft/hard とも失効抑止、本来の hard を過ぎていれば pin 切れで即 HardExpired、HardExpired への pin 不可、再 pin/unpin 可。Store 層 `pin_authenticated` は **再認証必須**（Active からでも要求、extend の非対称）、`unpin` は認証不要（安全側）。`AuthOperation::Pin` 追加、`kv.pin`/`kv.unpin` protocol + CLI、status/list に pin 残り秒。Alternatives: hard_deadline 直接ずらし案（pin の deadline 指定の方がユーザ意図に正確）/ soft だけ延長案（夜間 hard を満たさず却下）。DR-0004 authsock 移植の前提コア修正（Iteration -1）
